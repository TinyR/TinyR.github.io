应用程序配置

发表于 2017-06-24   |   分类于 后台开发

  应用程序离不开配置模块。数据库，日志等组件都需要进行配置。

配置信息来源

  1. 命令行
  2. 参数
  3. 环境变量
  4. 配置文件
  5. 默认值 default

  组件本身配置的默认值处于最低优先级，应用程序在启动的时候，读取配置信息，将会覆盖默认值。配置信息可以通过配置文件，环境变量，命令行参数等读取。

配置管理

运行环境的区分

  在不同的运行环境：生产环境(Production)、开发环境(Devlopment)、测试环境(Test)，应用程序需要读取不同的配置信息。

  可以设置机器的系统环境变量，应用程序编译运行时，通过检测当前系统环境变量来区分运行环境。

配置文件

  一种方案是将主要的配置信息集中存储在配置文件(比如：yaml,.properties)中，应用程序编译运行的时候读取配置文件，获取配置信息。

  1. 根据环境变量检测当前运行的环境，生产环境(Production)、开发环境(Devlopment)、测试环境(Test)
  2. 根据不同的运行环境读取不同的配置文件

保护生产环境的配置信息

  生产环境的一些配置信息，不应该完全暴露给普通开发人员。可以将生产环境配置文件存放在专门的服务器上，只有生产环境的机器才能够访问。应用程序在生产环境运行时，通过HTTP或者其他方式读取配置服务器上的配置信息。

配置中心

  简单的配置文件能够满足部分应用程序的需求。但存在一些不足，需要一个配置中心：集中管理依赖的配置、提供web管理平台进行配置和查询、支持应用程序拉取配置信息、支持发布与订阅，当配置变更时主动通知应用程序。

  有一些开源的解决方案，比如：阿里的diamond，应用程序不仅可以在启动时从diamond获取相关的配置，而且可以在运行中对配置数据的变化进行感知并获取变化后的配置数据。
